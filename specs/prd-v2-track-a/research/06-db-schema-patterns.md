## Conventions

**IDs:** `uuid("id").primaryKey().defaultRandom()` — PostgreSQL `gen_random_uuid()`. No nanoid/ULID/custom.

**Timestamps:** Always `timestamp("column", { withTimezone: true }).defaultNow().notNull()`. Both `createdAt` and `updatedAt` on every table.

**Enums:** No `pgEnum` — all stored as `varchar(20)` with documented valid values. This means new enum values don't require migrations.

**Foreign keys:** Explicit `.references()` with `onDelete` clause (`"cascade"` or `"set null"`).

**Indexes:** Composite indexes for query patterns. Named `idx_table_columns`. Unique indexes for one-to-one relationships.

**Prices:** `numeric(10, 2)` for money. Coordinates: `numeric(10, 7)`.

**Zod integration:** Each schema file exports `createSelectSchema()` and `createInsertSchema()` via `drizzle-zod`.

**Casing:** Drizzle configured with `casing: "snake_case"` — JS camelCase auto-converted to SQL snake_case.

## DB Connection (`src/shared/infra/db/drizzle.ts`)

- `postgres.js` driver (serverless-friendly)
- Global singleton with hot-reload protection
- Connection pooling: 5 (Vercel) or 10 (local)
- Schema passed for type safety

## Transaction Manager

**Interface** (`src/shared/kernel/transaction.ts`):
```ts
interface TransactionManager {
  run<T>(fn: (tx: TransactionContext) => Promise<T>): Promise<T>;
}
```

**Implementation** (`src/shared/infra/db/transaction.ts`): Wraps `db.transaction()`. Auto-rollback on exception.

**Usage:** Services accept optional `RequestContext` with `tx` field. Repos use `ctx?.tx ?? this.db` for queries.

## ID Generation

- **Primary keys:** UUID v4 at DB level (`gen_random_uuid()`)
- **Order numbers:** Sequential "ORD-XXXX" generated in repository via `MAX(CAST(REPLACE(...)))` query
- **Slugs:** Application-generated, varchar(200), unique
- **No short code utilities exist** — will need one for table `publicId` and ticket codes

## Migrations (`drizzle/`)

- Single initial migration: `0000_light_eddie_brock.sql`
- Pure SQL DDL with `--> statement-breakpoint` separators
- Generated by Drizzle CLI (`pnpm db:generate`)
- Applied via `pnpm db:migrate`

## Key Implications for New Schemas

1. **Follow the pattern:** uuid PK + timestamps + varchar enums + explicit FKs
2. **New `publicId` field for tables:** Need a short, URL-safe ID generator (nanoid or similar) — first non-UUID ID in the codebase
3. **New `ticketCode` for orders:** Human-readable code (e.g., "A-042") — needs a generation utility
4. **New `idempotencyKey` for orders:** Client-generated UUID, unique constraint
5. **Barrel export:** Add new schema files to `schema/index.ts`
6. **No pgEnum needed:** Continue varchar pattern for `tableSession.status` ('active' | 'closed')
