# New Product Status

> What the new version of CravingsPH delivers today.

## What Exists

The new version is an **authenticated shell** — it handles user identity but has no restaurant or menu functionality.

### User Account Management
- Register with email and password
- Log in with email/password, magic link (passwordless), or Google
- View and edit profile (display name, email, phone, avatar)
- Role-based access (admin, member, viewer — not yet used for anything)

### Route Structure
- Public home page
- Login, registration, and magic link pages
- Dashboard page (empty — placeholder)
- Profile settings page

## What Does NOT Exist

Everything from the legacy product is missing:

- Restaurant pages
- Menu browsing
- Item customization
- Cart
- Bill/receipt view
- Restaurant info display

And everything the legacy was also missing:

- Order placement
- Payments
- Restaurant management
- Discovery / search
- Notifications
- Analytics

## Summary

The new version solves the **"who is this user?"** question that the legacy ignored. But it delivers zero product value to either customers or restaurant owners today. It's a foundation, not a product.
